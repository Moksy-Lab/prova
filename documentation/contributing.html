Contributing

We welcome contributions to the Moksy e-learning templates. This section provides information on how you can contribute, including guidelines for submitting issues, the code of conduct, and ways to get involved.

How to Contribute

Contributing to the Moksy e-learning templates is simple. Here’s how you can get involved:

  1. Fork the repository: Start by forking the GitHub repository to your own account.
  2. Clone your fork: Clone the forked repository to your local machine using the following command: 
              
          git clone https://github.com/YOUR-USERNAME/moksy.git
  3. Create a new branch: Create a new branch for your feature or bug fix: 
              
          git checkout -b feature-name
  4. Make your changes: Implement your feature or bug fix in your local repository.
  5. Commit your changes: Commit your changes with a descriptive message: 
              
          git commit -m "Add feature name"
  6. Push to your fork: Push your changes to your forked repository: 
              
          git push origin feature-name
  7. Submit a pull request: Go to the original repository on GitHub and create a pull request from your forked branch.

For more detailed instructions, refer to the contributing guidelines in the repository.

Code of Conduct

To ensure a welcoming and inclusive community, we have adopted a code of conduct that participants are expected to adhere to. Please read the following guidelines carefully:

  • Be respectful and considerate to others.
  • Refrain from any form of harassment or discrimination.
  • Provide constructive feedback and be open to receiving it.
  • Focus on what is best for the community and the project.

For the full code of conduct, refer to the code of conduct document in the repository.

Submitting Issues

If you encounter any bugs or have suggestions for improvements, you can submit an issue. Here’s how to do it:

  1. Check for existing issues: Before submitting a new issue, check the issue tracker to see if someone has already reported the same issue.
  2. Open a new issue: If your issue has not been reported, open a new issue by clicking the "New issue" button in the repository's issues section.
  3. Provide a clear description: Provide a clear and concise description of the issue. Include steps to reproduce the problem, if applicable.
  4. Include relevant details: Include any relevant details, such as screenshots, error messages, and the environment in which the issue occurred.
  5. Submit the issue: Once you have filled out the issue template, submit the issue and wait for feedback from the maintainers.

For more detailed instructions, refer to the issue submission guidelines in the repository.
